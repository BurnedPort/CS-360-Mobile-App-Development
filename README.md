# CS-360-Mobile-App-Development

App Requirements and Goals
The goal of this project was to design and develop a functional inventory tracking mobile application. The app was created to help users organize and manage inventory items efficiently by allowing them to store, view, and update item information in a structured way. The primary user need addressed by this app was simplifying inventory management through a mobile interface that makes tracking items quick and accessible.

User Needs and UI Design
To support user needs, the application included screens for viewing inventory items, adding new entries, and managing stored data. The UI design focused on clarity and usability by keeping layouts simple, using clear labels, and minimizing the number of steps required to complete tasks. Navigation between screens was designed to feel predictable so users could quickly understand how to interact with the app. These design choices helped create a user-centered experience that prioritized efficiency and ease of use.

Development Approach
I approached development incrementally by building core features first and then expanding functionality step by step. I focused on modular code organization and separating UI behavior from data handling logic where possible. Frequent testing during development helped confirm each feature worked before moving forward. This iterative strategy made debugging easier and is a process I plan to continue using in future software projects.

Testing and Validation
Testing was performed throughout development by running the application regularly, validating user inputs, and confirming that navigation and inventory updates worked correctly. This process ensured the application behaved consistently and helped identify logic or interface issues early. Testing is important because mobile applications rely heavily on user interaction, and even small errors can negatively impact usability.

Innovation and Problem Solving
One challenge was translating the initial UI design into a functional application while maintaining usability across different screens. Adjustments were required to ensure layouts displayed correctly and that data updates were reflected properly within the app. Overcoming this required refining both the interface and underlying logic to align functionality with the intended user experience.

Areas of Success
I was particularly successful in connecting UI design with working application functionality. The inventory tracker demonstrates my ability to move from planning and design concepts to a fully functioning mobile application while applying user-centered design principles and mobile development best practices.
